Step 1: FSM 모델링

디지털 회로를 수학적 객체로 변환 — SEC의 출발점

FSM이란?

디지털 회로는 클럭마다 상태가 바뀌는 기계입니다. 이걸 수학으로 표현한 것이 FSM(Finite State Machine)입니다.

M = ( S, s₀, I, O, δ, λ )
기호의미회로에서의 대응
S상태 집합모든 FF 값의 가능한 조합
s₀초기 상태리셋 후 FF 값
I입력 집합primary input 포트들
O출력 집합primary output 포트들
δ: S × I → S상태 전이 함수조합 로직 + FF의 D 입력
λ: S × I → O출력 함수조합 로직 → output 포트

예시: 2-bit Counter

SystemVerilog

always_ff @(posedge clk)
  if (!rst_n) cnt <= 2'b00;
  else       cnt <= cnt + 1;

assign out = (cnt == 2'b11);

FSM 표현

S= {00, 01, 10, 11}
s₀= 00 (리셋)
δ: 00→01→10→11→00
λ: 11→1, 나머지→0

상태 전이도

cnt:00
out:0

상태 폭발 (State Space Explosion)

FF 수가 늘어나면 상태 공간이 지수적으로 커집니다. 이것이 SEC를 어렵게 만드는 근본 원인입니다.

FF 개수상태 수전수조사 가능?
2개4가능
8개256가능
16개65,536힘듦
32개~4.3 × 10⁹불가능
320개~2 × 10⁹⁶절대 불가능

FF 수에 따른 상태 공간 (로그 스케일)

실제 회로: FF 수천~수만 개
→ 모든 상태를 직접 나열하는 것은 불가능 → BDD, SAT가 필요한 이유 (4~5단계)

λ 함수의 두 가지 형태: Moore vs Mealy

FSM 6-tuple 중 출력 함수 λ를 어떻게 정의하느냐에 따라 두 모델로 나뉩니다.

핵심 차이: 출력이 뭘 보는가?

Moore

출력 = f( 상태 )
λ: S → O
Mealy

출력 = f( 상태, 입력 )
λ: S × I → O

Moore  신호등 — 상태가 출력을 결정

버튼을 눌러도 상태가 바뀌기 전까지 출력은 고정됩니다.

빨간불 출력: 정지
초록불
현재 상태:빨간불
출력:정지
입력과 무관 — 상태만 보고 출력 결정

출력이 상태 안에 적혀있음 (원 안)

Mealy  자판기 — 상태 + 입력이 출력을 결정

같은 대기 상태에서 넣는 금액에 따라 출력이 달라집니다.

동전을 넣어주세요
상태:대기
잔액:0원
입력:-
출력:-
같은 "대기" 상태 — 입력에 따라 출력이 즉시 달라짐

출력이 화살표 위에 적혀있음 (전이에 의존)

왜 Mealy가 상태 수가 더 적은가?

같은 동작 "req 오면 gnt 내보내기"를 두 방식으로 구현해봅시다. 클릭해서 비교하세요.

Moore  상태 2개 필요

상태:IDLE
gnt:0

gnt=1을 내보내려면 GRANT 상태로 이동해야 함
→ 출력이 다르면 상태도 달라야 한다

Mealy  상태 1개로 충분

상태:IDLE
gnt:0

상태 이동 없이 입력만 보고 즉시 출력 결정
→ GRANT 상태가 필요 없다

극단적 예시: 출력 4가지인 시스템

S0
out=A
S1
out=B
S2
out=C
S3
out=D

Moore: 최소 4개 상태

출력마다 전용 상태 필요

S0
i=00 → out=A i=01 → out=B i=10 → out=C i=11 → out=D

Mealy: 1개 상태로 충분

입력 조합으로 출력 구분

복잡도가 사라지는 게 아니라
상태(원)에서 전이(화살표)로 이동하는 것

비교

Moore Mealy
출력 결정상태만상태 + 입력
출력 함수λ: S → Oλ: S × I → O
출력 변경클럭 edge (1 cycle 지연)입력 변화 즉시

Moore는 Mealy의 특수한 경우
λ(s, i) 에서 i를 무시하면 → λ(s) = Moore

SEC에서 Mealy로 통일하는 이유

실제 회로는 registered output과 combinational output이 섞여있습니다.
Mealy 모델(λ: S × I → O)이 두 경우를 모두 표현할 수 있으므로,
SEC 도구는 Mealy로 통일합니다.

이것이 Miter 구성에서 중요한 이유: 같은 상태 쌍 (sR, sI)이라도
어떤 입력을 넣느냐에 따라 출력이 달라질 수 있기 때문입니다.
