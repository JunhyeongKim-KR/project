Step 0: 왜 SEC를 쓰는가?

Formal Verification의 세 기둥과 SEC의 위치

Formal Verification의 세 가지 질문

Formal Verification은 세 가지 근본 질문에 답합니다. 각 카드를 클릭하면 상세 내용이 펼쳐집니다.

P
FPV
Property Verification
"설계 A ⊨ 속성 P?"
이 설계가 올바른가?
입력: 설계 1개 + SVA assertions
찾는 것: 설계 자체의 버그
비유: "시험 답안이 정답인가?" (정답지 = SVA)
FPV는 설계에 대해 수학적 속성(assertion)이 항상 성립하는지 증명합니다.

예: assert property (@(posedge clk) req |-> ##\[1:3\] gnt);
"req가 오면 1~3 사이클 내에 gnt가 반드시 나가야 한다"

모든 가능한 입력 시퀀스에 대해 검증하므로, 시뮬레이션으로 잡지 못하는 코너 케이스 버그를 찾아냅니다.
클릭하여 자세히 보기
L
LEC
Logic Equivalence Checking
"설계 A == 설계 B?"
FF 구조 동일 전제
입력: 설계 2개 (변환 전/후)
찾는 것: 변환이 조합 로직을 바꿨는가
비유: "번역본이 원본과 같은가?" (단어 대 단어 비교)
LEC는 두 설계의 FF를 1:1 매칭한 뒤, 각 FF 쌍 사이의 조합 로직이 동일한지 비교합니다.

합성(Synthesis) 후 게이트 넷리스트가 원래 RTL과 같은 기능을 하는지 확인하는 데 주로 사용됩니다.

한계: FF 구조가 바뀌면 매칭 자체가 실패합니다. Retiming이나 clock gating처럼 FF 위치가 바뀌는 변환에는 사용할 수 없습니다.
클릭하여 자세히 보기
S
SEC
Sequential Equivalence Checking
"설계 A == 설계 B?"
FF 구조 달라도 OK
입력: 설계 2개 (구조 변환 전/후)
찾는 것: 구조 변환이 기능을 바꿨는가
비유: "번역본이 원본과 같은가?" (의미 전체 비교, 문장 구조 달라도 OK)
SEC는 FF 매칭 없이 입출력 동작(I/O behavior)만 비교합니다.

동일한 입력 시퀀스를 주었을 때 동일한 출력 시퀀스가 나오면 Equivalent입니다.

Retiming, clock gating, pipeline 추가, FSM re-encoding 등 FF 구조가 바뀌는 모든 변환에 사용할 수 있습니다.

대신: LEC보다 훨씬 어렵고 느립니다. FF 구조가 보존되는 경우엔 LEC를 쓰는 것이 효율적입니다.
클릭하여 자세히 보기

언제 LEC가 안 되고 SEC가 필요한가?

Retiming 변환을 예시로 봅시다. FF 위치가 바뀌면 LEC는 매칭에 실패합니다.

Before: 원래 RTL

FF_A → comb logic → FF_B
FF가 로직의 양쪽 끝에 위치

After: Retiming 후

comb part1 → FF_A' → comb part2 → FF_B'
FF가 조합 로직 사이로 이동

검증 결과

LEC FAIL
FF_A ↔ FF_A' 매칭 시도
→ 위치가 다름 → 조합 로직 경계 불일치
→ 비교 불가능
SEC PASS
FF 매칭 불필요
→ 같은 입력 → 같은 출력?
→ I/O 동작 동일 확인

변환 유형별 비교

변환 FF 구조 LEC SEC
합성 (기본) 동일 O O
Retiming 변경 X O
Clock gating 변경 X O
Pipeline 추가 변경 X O
FSM re-encoding 변경 X O

SEC의 한계

⚠ 핵심 주의
SEC는 두 설계가 같은지만 봅니다.
둘 다 틀려도 Equivalent입니다.

버그가 숨는 시나리오

설계 A ✖ 버그 있음
sum = a + b;  // carry bit 누락!
↓ 구조 변환
설계 B ✖ 같은 버그 유지
sum = a + b;  // carry bit 여전히 누락!
SEC 결과: "Equivalent" ✓
← 버그를 못 잡음!

왜 FPV가 먼저인가

FPV 설계가 올바른지 확인
"carry bit가 포함되어야 한다"라는 assertion을 검증
→ FAIL → 버그 발견!
SEC 변환이 정확한지 확인
FPV로 올바름을 확인한 뒤,
변환이 그 올바름을 보존했는지 확인
SEC = "같은가?"
"맞는가?"는 FPV의 영역
