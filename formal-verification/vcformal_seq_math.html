디지털 회로 → FSM 모델링

모든 synchronous 디지털 회로는 Mealy FSM으로 수학적으로 정확하게 모델링됩니다.

Step 1 상세 시각화: FSM 모델링 & Moore vs Mealy →
M = ( S, s₀, I, O, δ, λ )

S : 유한 상태 집합   // 모든 레지스터 조합 = 2ⁿ개 상태
s₀ : 초기 상태 (reset state)
I : 입력 알파벳 (primary inputs)
O : 출력 알파벳 (primary outputs)
δ : S × I → S   // 상태 전이 함수 (next-state logic)
λ : S × I → O   // 출력 함수 (output logic)
핵심 통찰: 32-bit 레지스터 10개만 있어도 상태 공간은 2³²⁰ ≈ 10⁹⁶개. 시뮬레이션으로 전수 검사는 물리적으로 불가능합니다.

Combinational Logic

상태 없음 → Boolean function으로 표현
입력 → 출력 직접 매핑
f: {0,1}ⁿ → {0,1}ᵐ

Sequential Logic (FSM)

상태 있음 → 시간에 따라 동작 변화
같은 입력이라도 상태에 따라 출력 다름
λ(s, x) — 상태 s에 의존

Sequential Equivalence 정의

두 FSM R (Reference)과 I (Implementation)의 등가성을 수학적으로 정의합니다.

Definition: R ≡seq I  ⟺

  ∀ input sequence  x₁, x₂, ..., xₖ  (∀ k ∈ ℕ):

    λR(sR(k), xₖ) = λI(sI(k), xₖ)

// 여기서 상태는 재귀적으로 정의:
  sR(0) = s₀R   (reset 상태)
  sR(k+1) = δR(sR(k), xₖ)   (전이)
CEC와의 차이: Combinational EC는 레지스터를 1:1 매핑한 뒤 사이의 조합 로직만 비교합니다. SEQ는 레지스터 매핑 없이 I/O 동작 자체를 비교하므로 retiming, FSM 재인코딩에도 적용 가능합니다.
비교 방식CEC (Combinational)SEQ (Sequential)
비교 대상조합 로직 cone전체 I/O 동작
레지스터 매핑필수 (1:1)불필요
Retiming처리 불가처리 가능
복잡도coNP-completePSPACE-complete

Product Machine 구성

두 FSM을 하나의 곱 기계(Product Machine)로 결합하여, 등가성 문제를 도달가능성 문제로 변환합니다.

Product Machine  P = R × I

상태 공간:  SP = SR × SI
초기 상태:  s₀P = (s₀R, s₀I)
전이 함수:  δP((sR, sI), x) = (δR(sR, x), δI(sI, x))
Miter 출력:  miter(sR, sI, x) = λR(sR, x) ⊕ λI(sI, x)
안전 속성:  ∀ reachable (sR, sI), ∀x:  miter(sR, sI, x) = 0

핵심 변환:
  R ≡seq I  ⟺  ∀ reachable states, ∀ inputs: miter = 0
문제 변환 완료: "두 회로가 같은가?" → "Product machine에서 Bad 상태에 도달 가능한가?" 이것은 formal verification의 고전적인 safety property checking 문제입니다.

Reachability Analysis

Bad 상태에 도달 가능한지를 판별합니다. 상태를 개별 열거하면 폭발하므로, 심볼릭(symbolic) 방법을 사용합니다.

Fixed-Point Iteration (BDD 기반):

Reach₀(s) = Init(s)   // 초기 상태 집합
Reachₖ₊₁(s) = Reachₖ(s) ∪ Image(Reachₖ, δ)

// Image 연산 (후속 상태 계산):
Image(R, δ) = { s' | ∃s ∈ R, ∃x ∈ I: δ(s, x) = s' }

종료 조건: Reachₖ₊₁ = Reachₖ  // 고정점 (fixed point)

판정:
  Reach* ∩ Bad = ∅  → EQUIVALENT (PASS)
  Reach* ∩ Bad ≠ ∅  → NOT EQUIVALENT (FAIL)
BDD (Binary Decision Diagram): 상태 집합을 Boolean function으로 압축 표현합니다. 2¹⁰⁰개 상태도 하나의 BDD 노드 그래프로 표현 가능하여, 개별 열거 없이 집합 연산을 수행합니다.

SAT 기반 검증 & k-Induction

BDD의 메모리 한계를 극복하기 위해, 현대 SEQ 도구는 SAT solver와 수학적 귀납법을 결합합니다.

BMC (Bounded Model Checking)

// 회로를 k 사이클 unroll:

Init(s₀)
∧ T(s₀, i₀, s₁)
∧ T(s₁, i₁, s₂)
∧ ...
∧ T(sₖ₋₁, iₖ₋₁, sₖ)
∧ (Bad(s₀) ∨ Bad(s₁) ∨ ... ∨ Bad(sₖ))

UNSAT → k 내 모든 step 안전
SAT → 반례 발견!

k-Induction (완전 증명)

// Base Case (SAT query → UNSAT 확인):
Init(s₀) ∧ T(s₀,i₀,s₁) ∧ ... ∧ T(sₖ₋₁,iₖ₋₁,sₖ)
∧ (Bad(s₀) ∨ ... ∨ Bad(sₖ))

// Inductive Step (SAT query → UNSAT 확인):
¬Bad(s₀) ∧ T(s₀,i₀,s₁) ∧ ¬Bad(s₁)
∧ ... ∧ T(sₖ₋₁,iₖ₋₁,sₖ)
∧ Bad(sₖ)

둘 다 UNSAT → 완전 증명!
Auxiliary Invariant가 필요한 이유:
Inductive step은 임의 상태에서 시작 — 실제로 도달 불가능한 상태도 포함됨.
도달 불가능한 상태에서 Bad가 성립하면 false counterexample 발생.

해결: Inv(s)를 추가하여 unreachable 상태 제외:
¬Bad(s₀) ∧ Inv(s₀) ∧ T(s₀,i₀,s₁) ∧ ... → ¬Bad(sₖ) ∧ Inv(sₖ)

k를 늘리면 "k개 연속 ¬Bad" 조건이 unreachable 상태를 자연 제외.
그래도 실패 시 → interpolation으로 invariant 자동 생성.
BMC만으로는 부족: k=100까지 UNSAT이어도 k=101에서 반례가 있을 수 있습니다. k-Induction은 "k개 연속 안전 상태 → 다음도 안전"을 증명하여 모든 k에 대해 보장합니다.
기법완전성장점단점
BDD Fixed-Point완전고정점 = 완전 증명메모리 폭발
BMC (SAT)불완전반례 빠르게 발견bound 내만 검증
k-Induction완전SAT 기반 + 완전 증명invariant 필요할 수 있음
Interpolation완전overapprox.으로 귀납 강화수렴 보장 없음

Pipeline SEC: Latency Offset

Retiming/Pipeline 변환 시 출력 latency가 달라집니다. SEC는 이를 수학적으로 보정합니다.

// Spec: latency = LR,  Impl: latency = LI
// Offset N = LI - LR

Latency-Adjusted Equivalence:

∀k ≥ 0, ∀ input sequence x₀, x₁, ...:
  λR(sR(k), xk) = λI(sI(k+N), xk+N)

// 즉, Impl의 출력을 N cycle 뒤에서 비교

예시: 2-stage → 3-stage (N=1)

Cycle: 1 2 3 4 5 6
Spec: · · R1 · · R2
Impl: · · · R1 · · R2
             ↕
          offset=1

Miter에서의 처리

// Impl 출력에 N-cycle shift register 삽입:

oI_delayed = delay(oI, N)

miter = oR ⊕ oI_delayed

// 또는 Spec에 N-cycle 추가 입력 파이프라인
// VCF TCL: set_seq_equiv_latency -latency N
적용 대상: Retiming (FF 위치 이동), Pipeline stage 추가/제거, Clock gating (latency 변화 없으나 동작 조건 변경)

최적화 기법 & 전체 흐름

실제 산업용 회로에 적용하기 위한 핵심 최적화 기법들입니다.

기법원리효과
Structural Hashing 구조적으로 동일한 게이트/서브회로를 먼저 매칭 검증 범위 50~90% 축소
Signal Correspondence 내부 등가 신호쌍을 simulation + SAT로 발견 상태 공간 대폭 축소
CEGAR Counterexample-Guided Abstraction Refinement 추상화 → 반례 검사 → 정밀화 반복
Interpolation UNSAT proof에서 overapproximate 도달 집합 추출 귀납 보조 정리(invariant) 자동 생성
Phase Abstraction multi-clock 설계에서 위상별 분리 검증 multi-clock retiming 처리
최종 정리:
회로 → FSM → Product Machine → Reachability → SAT/Induction → 수학적 완전 증명
