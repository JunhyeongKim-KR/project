DiskANN Full Pipeline

Vamana Graph + PQ + Multi-Query Search

Step 1 — Vamana Graph 구조

DiskANN은 데이터를 Vamana 그래프로 구성합니다. 각 노드는 벡터, 엣지는 가까운 이웃 연결입니다. 탐색 시작점(medoid)에서 greedy search로 목표에 접근합니다.

Vamana 그래프 특징:
- 각 노드의 이웃 수(degree)는 R로 제한 (보통 R=64~128)
- Medoid (중심점)에서 시작하여 greedy하게 가까운 이웃으로 이동
- HNSW와 달리 단일 레이어로 구성 → 디스크 저장에 유리

Step 2 — PQ Codebook (메모리 상주)

각 노드의 벡터를 PQ로 압축하여 메모리에 올립니다. 원본 벡터+그래프 엣지는 SSD에 저장됩니다.

메모리 구성: PQ 코드 (N개 벡터 x M bytes) + Codebook (M x K x d/M x 4B)
SSD 구성: 원본 벡터 + 그래프 인접 리스트 (노드별 R개 이웃 ID)

서브스페이스 분할, K-Means Codebook 생성 과정, PQ 인코딩, LUT/ADC 거리 계산을 단계별로 확인할 수 있습니다.

Step 3 — Multi-Query LUT 생성

쿼리가 여러 개 동시에 들어오면, 각 쿼리마다 독립적으로 LUT를 생성합니다. LUT 생성은 쿼리당 1회, codebook 크기에만 의존합니다.

Step 4 — Graph Search (Beam Search)

각 쿼리가 medoid에서 시작하여 그래프를 탐색합니다. PQ+LUT로 이웃 거리를 빠르게 계산하고, beam width만큼 후보를 유지하며 전진합니다.

Step 5 — SSD Rerank

Graph search로 각 쿼리의 Top-K 후보를 찾은 뒤, SSD에서 원본 벡터를 읽어 정확한 거리를 계산합니다.

Step 6 — Full Pipeline 비교
