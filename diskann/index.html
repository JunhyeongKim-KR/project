Product Quantization (PQ)

벡터 압축 & Codebook 생성 시각화

Step 1 — 원본 벡터

D=16 차원의 float32 벡터 8개가 있습니다. 각 셀은 하나의 차원 값입니다.

각 벡터: 16차원 × 4바이트(float32) = 64 bytes
전체 8개 벡터: 512 bytes

Step 2 — 서브스페이스 분할

16차원 벡터를 M=4개의 서브벡터(각 4차원)로 분할합니다. 같은 색이 같은 서브스페이스입니다.

분할 규칙: 연속된 차원을 순서대로 M등분
dim [0-3] → 서브스페이스 1, dim [4-7] → 서브스페이스 2, dim [8-11] → 서브스페이스 3, dim [12-15] → 서브스페이스 4

Step 3 — K-Means로 Codebook 생성

각 서브스페이스에서 독립적으로 K-means 클러스터링을 수행합니다. 2D로 투영하여 시각화합니다.

반복: 0 / 15
● 작은 점 = 서브벡터 데이터 포인트   ★ 큰 점 = centroid (codebook entry)
K=4 (시각화 단순화 목적, 실제로는 K=256이 일반적)

Step 4 — PQ 인코딩

각 서브벡터를 가장 가까운 centroid의 인덱스로 치환합니다. 아래 Codebook에서 각 점이 어떤 centroid에 매핑되는지 확인하세요.

Codebook (Step 3 결과)

인코딩: 각 서브벡터 → 가장 가까운 centroid 찾기 → 그 인덱스(0~3)를 저장
원본 16 × float32 = 64 bytes → PQ 코드 4 × uint8 = 4 bytes

Step 5 — 거리 계산 (ADC)

쿼리 벡터가 들어오면, 미리 lookup table을 만들어 빠르게 거리를 계산합니다.

다양한 설정에서의 PQ 압축 효과를 비교합니다.
