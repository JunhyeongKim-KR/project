Cascade HD란?

HD를 한 번만 적용하면 저차원(d=2,4,8)에서는 편향이 남습니다.
Cascade: 사전에 고정한 랜덤 부호 D1~Dk로 HD를 반복 적용하면, 적은 비용(O(d log d) × k)으로 R에 가까운 균등화를 달성할 수 있습니다.

HD×1: x̃ = (1/√d) H D1 x
HD×2: x̃ = (1/√d) H D2 (1/√d) H D1 x
HD×3: x̃ = (1/√d) H D3 (1/√d) H D2 (1/√d) H D1 x
HD×4: x̃ = (1/√d) H D4 ··· (1/√d) H D1 x

D1~Dk는 차원당 한 번만 생성되며, 모든 샘플에 동일하게 적용됩니다.

관찰 포인트

1. d=2~8에서 cascade 횟수가 늘수록 KS 거리가 어떻게 줄어드는가
2. d=64 이상에서는 1회만으로도 R과 동일한지 확인
3. 몇 회 cascade면 R 수준에 도달하는가 — 비용 대비 효과의 sweet spot

각 블록의 그림 읽는 법

각 d마다 R 1열 + HD 4열 = 5열, scatter + histogram 2행:

입력 scatter — (x[0], x[1]) 투영. 편향이 클수록 x축에 몰림
R출력 scatter — (Rx)[0,1] 투영. 랜덤 회전 결과 — 항상 완벽히 균등
HD출력 scatter — (HDx)[0,1] 투영. d가 작으면 구조 남음, d가 크면 R과 동일

아래 행은 각각의 첫 번째 좌표 히스토그램. 빨간 곡선 = 이론 정규분포 N(0, 1/d)

KS 거리란?

KS (Kolmogorov-Smirnov) 거리 = 히스토그램(실측)과 빨간 곡선(이론 정규분포) 사이의 최대 간격.
두 분포가 얼마나 다른지 하나의 숫자로 요약합니다.

KS < 0.015 → 거의 동일 (OK)  |  KS 0.015~0.04 → 경계 (borderline)  |  KS > 0.04 → 눈에 보일 정도로 다름 (NOT OK)

R vs HD 핵심 차이

R (랜덤 직교행렬)은 어떤 차원에서든 완벽한 균등화를 보장합니다. d=2에서도 완벽합니다. 대신 O(d²) 비용.

HD (Hadamard)는 d가 작으면 근사이지만, d ≥ 64에서는 R과 구별 불가능해집니다. 비용은 O(d log d).

실제 LLM의 head dimension은 128 이상이므로, HD로 R을 완벽히 대체 가능합니다.
