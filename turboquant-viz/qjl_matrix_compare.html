QJL 행렬 방법 비교

QJL의 핵심 조건: E\[Sij\] = 0, E\[Sij²\] = 1
세 가지 행렬이 이 조건을 만족하는지 수학적·실험적으로 검증

방법 수식 연산량 E\[Sij\] = 0 E\[Sij²\] = 1 분산 보장
원래 N(0,1) Sij ~ N(0,1) O(d²) ✅ 정확 ✅ 정확 ✅ 정확
SRHT S = (1/√k) P H D O(d log d) ✅ ✅ ✅ 근사
Rademacher Sij ~ {-1, +1} O(d²) 빠른 상수 ✅ ✅ ✅
N(0,1) Sij ~ N(0,1)
✅ 불편성 증명
E\[Sij\] = 0 (정규분포 대칭)
E\[Sij²\] = Var + μ² = 1+0 = 1
→ √(π/2) 보정이 정확히 성립
✅ 분산 상한
Var\[⟨y, r̂⟩\] ≤ (π/2d)‖y‖²
→ 가우시안 회전 불변성 덕분
→ 이론값 = 실측값
SRHT S = (1/√k) P H D
✅ 구성 요소
D : ±1 대각행렬 → E\[Dii\]=0, E\[Dii²\]=1
H : Hadamard → O(d log d)
P : k행 랜덤 샘플링
→ 두 조건 모두 만족
⚠️ 분산이 '근사'인 이유
S의 행들이 완전 독립이 아님
(H의 구조적 상관관계 존재)
→ 분산 상한이 상수 배 느슨
→ 실용상 무시 가능
Rademacher Sij ~ {-1,+1}
✅ 두 조건 만족
E\[Sij\] = (+1-1)/2 = 0 ✓
E\[Sij²\] = (1²+1²)/2 = 1 ✓
→ 불편성 정확히 유지
✅ 왜 '빠른 상수'
곱셈 없이 부호 반전만:
Sij × rj → ±rj
→ float 곱셈 대비 수배 빠름
→ O(d²)이지만 상수 매우 작음

3. 추정 오차 (RMSE) — 차원별 비교

d가 커질수록 세 방법의 RMSE가 어떻게 수렴하는가

결론

세 방법 모두 E\[Sij\]=0, E\[Sij²\]=1을 만족하므로 QJL의 불편 추정량 조건을 충족합니다.
SRHT: O(d log d)로 가장 빠름. 분산 상한이 약간 느슨하지만 실용상 차이 없음.
Rademacher: 구현 최간단. 곱셈 없이 부호 반전만으로 연산. 하드웨어 친화적.
N(0,1): 이론적 기준선. 실제 배포에서는 SRHT나 Rademacher로 대체 권장.
