rtl-guard

Claude Code에서 rtl/ 폴더 접근을 원천 차단하는 3겹 보안 구조

1
CLAUDE.md - 규칙 인식 SOFT

Claude가 세션 시작 시 읽고 자발적으로 따르는 규칙. "rtl/ 접근 금지"라고 적혀있으면 스스로 안 함.

~/CLAUDE.md
2
.claudeignore - 검색 차단 MEDIUM

Glob/Grep 검색 결과에서 rtl/ 하위 파일을 제외. Claude가 검색으로 rtl/ 파일을 "발견"하는 것 자체를 방지.

~/.claudeignore
3
PreToolUse Hook - 도구 차단 HARD

모든 도구(Read/Edit/Write/Bash/Grep/Glob) 호출 시 경로/명령어에 rtl/이 포함되면 실행 자체를 거부. 최후의 방어선.

~/.claude/scripts/block-rtl-access.sh ~/.claude/settings.json
+
OS 파일 잠금 (선택) OS

chattr +i / chflags schg로 hook 파일 자체를 수정 불가능하게 잠금. root만 해제 가능.

sudo chattr +i (Linux) / sudo chflags schg (macOS)

실행 흐름

Claude: Read rtl/top.v
|
CLAUDE.md: "rtl/ 접근 금지" 규칙 인식
실수로 통과할 수 있음
|
.claudeignore: 검색 결과에서 제외
직접 경로 지정 시 통과
|
settings.json: PreToolUse hook 발동
|
block-rtl-access.sh: 경로에서 rtl/ 감지
|
BLOCKED (exit 2) - 도구 실행 거부

공격 시나리오별 차단 여부

시도Layer 1

Edit rtl/top.v차단통과차단
hook 파일 삭제 시도통과통과차단 (자기 보호)

설치

# 설치
bash install-rtl-guard.sh

# 임시 허가 (필요시)
touch ~/.rtl-guard-unlock
rm ~/.rtl-guard-unlock

sudo chattr +i ~/.claude/scripts/block-rtl-access.sh
sudo chattr +i ~/.claude/settings.json
sudo chattr +i ~/.claudeignore
